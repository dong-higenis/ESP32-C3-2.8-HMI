# Battery Monitoring System - Code Review Guide

## 프로젝트 개요

ESP32 기반 배터리 잔량 모니터링 시스템이다.
ADC로 분압된 배터리 전압을 읽고, OCV 룩업 테이블과 선형 보간으로 퍼센트를 환산하며, EMA 필터를 거쳐 LCD에 표시한다.

테스트 1회에 4시간 이상 소요되므로, 코드 리뷰 단계에서 최대한 많은 문제를 사전에 잡아야 한다.

## 핵심 파일

- `battery.cpp` — 배터리 전압 읽기, 퍼센트 환산, 필터링, 충전 감지, 셧다운 판단
- `battery.h` — 상수 정의 (OCV 테이블 인덱스, 핀, 임계값 등)
- `hw_def.h` — 하드웨어 핀 정의, 분압비, 캘리브레이션 오프셋, EMA 알파 등

## 리뷰 시 반드시 점검해야 할 항목

### 1. OCV 테이블 정합성

- 테이블이 전압 내림차순으로 정렬되어 있는지 확인할 것
- 퍼센트도 전압과 같은 방향으로 내림차순인지 확인할 것
- 중복 전압이나 역전된 구간이 없는지 확인할 것
- 선형 보간 시 분모 `(v_high - v_low)`가 0이 되는 구간이 없는지 확인할 것

### 2. 선형 보간 로직 (`voltageToPercent`)

- 테이블 범위 밖 전압에 대한 클램핑이 정확한지 확인할 것
- for 루프에서 `voltage >= v_low` 조건이 올바른 구간을 찾는지, 경계값에서 어떤 구간에 매칭되는지 확인할 것
- 보간 공식 `p_low + (voltage - v_low) / (v_high - v_low) * (p_high - p_low)`이 수학적으로 올바른지 확인할 것

### 3. EMA 필터

- `BAT_EMA_ALPHA` 값이 적절한지 평가할 것 (너무 높으면 노이즈 통과, 너무 낮으면 반응 지연)
- `filtered_percent` 초기값이 `-1.0f`인데, `batteryInit()`에서 올바르게 초기화되는지 확인할 것
- `batteryInit()`과 `batteryUpdate()` 양쪽에서 EMA 초기화 경로가 충돌하지 않는지 확인할 것

### 4. 충전 감지 및 오프셋 보정

- `BAT_CHARGE_VOLTAGE_OFFSET` 값이 과도하지 않은지 평가할 것
- 충전 중 오프셋을 뺀 전압이 OCV 테이블 하한 아래로 떨어질 가능성이 있는지 확인할 것
- 충전 ↔ 방전 전환 시 `filtered_percent`에 급격한 불연속이 발생하는지 확인할 것
- `was_charging` 상태 전환 시 별도 처리가 필요한지 평가할 것

### 5. 정수 변환 및 출력 안정성

- `(int8_t)(filtered_percent + 0.5f)` 반올림이 경계값(예: 99.5 → 100, -0.3 → 0)에서 올바르게 동작하는지 확인할 것
- `int8_t` 범위(-128~127) 내에서 퍼센트 값(0~100)이 항상 유지되는지 확인할 것
- 0% 미만이나 100% 초과 값이 display_percent에 들어갈 가능성이 있는지 확인할 것

### 6. ADC 읽기 및 전압 역산

- `readVoltage()` 함수에서 `analogReadMilliVolts` → 분압비 역산 → 캘리브레이션 오프셋 순서가 올바른지 확인할 것
- `BAT_DIVIDER_RATIO`와 `BAT_CALIBRATION_OFFSET` 값이 실측과 일치하는지 평가할 것
- 부팅 직후 ADC 불안정 구간에 대한 대비가 있는지 확인할 것

### 7. 셧다운 로직

- `BAT_SHUTDOWN_VOLTAGE` 임계값이 OCV 테이블 최하단과 정합성이 있는지 확인할 것
- 셧다운 판단 후 복귀 경로가 없는 것이 의도된 것인지 확인할 것
- 노이즈로 인해 임계값을 순간적으로 하회할 때 오작동 가능성이 있는지 확인할 것

### 8. 엣지 케이스

- 배터리를 아예 분리한 상태에서 ADC가 어떤 값을 읽는지, 그때 로직이 어떻게 동작하는지 확인할 것
- 충전 완료(4.2V 이상) 상태에서 퍼센트가 100%를 초과하지 않는지 확인할 것
- `batteryUpdate()`가 호출되지 않는 상황(loop 블로킹 등)에서의 영향을 확인할 것

## 과거 발생했던 버그 (참고)

단조성(monotonicity) 로직이 있었다. "충전 중에는 퍼센트 하락 차단, 방전 중에는 퍼센트 상승 차단" 규칙이었는데, 부팅 직후 ADC 불안정으로 초기 퍼센트가 비정상적으로 낮게 잡히면 이후 실제 퍼센트가 높아도 갱신이 영원히 차단되는 버그가 있었다. 현재는 해당 로직을 제거하고 EMA 필터 결과를 매 사이클 직접 반영하는 방식으로 수정되었다.

## 리뷰 결과 포맷

각 항목에 대해 다음 형식으로 보고할 것:

- **상태**: 정상 / 잠재적 위험 / 버그
- **설명**: 무엇을 확인했는지
- **근거**: 코드의 어떤 부분에서 판단했는지
- **제안**: (잠재적 위험 또는 버그인 경우) 구체적인 수정 방향
